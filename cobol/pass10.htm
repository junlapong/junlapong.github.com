การคอมไพล์ (compiling)การ RUN โปรแกรม
EDITOR กับการเขียนโปรแกรม
การคอมไพล์และการ RUN โปรแกรม
          จากที่ได้กล่าวมาแล้วว่า ภาษาโคบอลนั้นจริง ๆ แล้วไม่เหมาะที่จะใช้กับเครื่องระดับไมโครคอมพิวเตอร์ เนื่องจาก เครื่องไมโคร คอมพิวเตอร์มีขีดความสามารถที่จำกัดในเรื่องของหน่วยความจำ ถ้าหากโปรแกรมที่มีขนาดใหญ่มากจะมีปัญหาต่าง ๆ ที่ยุ่งยากตามมา แต่ก็มี Software ที่ถูกสร้างขึ้นโดยบริษัท Microsoft เป็นตัว Compile ที่ใช้สำหรับเครื่องไมโครคอมพิวเตอร์ ซึ่งในขณะนี้บริษัท Microsoft ได้ พัฒนาตัว COBOL Compiler ขึ้นเป็น Version 3.0 แล้ว แต่ตามตัวอย่างนี้ใช้ COBOL Compiler Version 2.2 อยู่ และโปรแกรมตัวอย่าง บางโปรแกรมในตำราเล่มนี้ สามารถ Compileด้วย COBOL Compiler Version 2.0 ขึ้นไป ซึ่งขั้นตอนการ Compile และการ RUN ไม่ แตกต่างไปกว่ากันมากนัก และมีขั้นตอนดังนี้
  1. สร้าง Source Program ลงในแผ่น Diskette หรือ Floppy Disk
  2. สร้างแฟ้มข้อมูลเพื่อเก็บข้อมูลเป็น Input Data ของโปรแกรมนี้ลงใน Diskette หรือ Floppy Disk เช่นเดียวกัน โดยตั้งชื่อ แฟ้มข้อมูลให้ตรงกับ Value of File-ID Clause ใน DATA DIVISION ของ Source Program ในข้อ 1
  3. เรียก COBOL Compiler มาใช้
EDITOR กับการเขียนโปรแกรม
          Editor ในความหมายที่เป็นภาษาไทยว่า "ตัวบรรณาธิกรณ์" ในที่นี้ผู้เขียนเรียกว่า "Editor" ดีกว่า Editor คือตัวที่ใช้แทนกระดาษ ทด กระดาษเขียน หรือจะเรียกว่าเป็นสมุดจดก็ว่าได้ หรือให้ความหมายได้เช่นเดียวกัน หรือที่เราใช้แทนกระดาษเขียนโปรแกรมนั้นเอง Editor เป็น ตัวที่ทำการประมวลผลกลุ่มตัวอักขระต่าง ๆ ซึ่งใช้พิมพ์จดหมาย พิมพ์รายงาน และเอามาเขียน หรือสร้าง Source Program หรือที่เรารู้จักกัน ในชื่อของ Word Processor เป็นที่น่าแปลกว่า มีผู้คนทั่วไปที่เกี่ยวข้องกันในทางคณิตศาสตร์ โดยเฉพาะไมโครคอมพิวเตอร์มักจะรู้จักหรือได้ ยินคำว่า "Word Processor" แต่ไม่มีใคร หรือน้อยคนนักที่จะรู้จักหรือได้ยินคำว่า "Editor" Software ที่เป็นประเภท Word Processor นั้น ในระดับไมโครคอมพิวเตอร์เป็นที่รู้จักกันดีก็คือ Wordstar , Word ราชวิถี เป็นต้น ส่วนที่นักศึกษาคอมพิวเตอร์รู้จักกันและใช้กันอยู่เสมอก็คือ TURBO PASCAL, TURBO-C, SIDEKICK เป็นต้น
          สำหรับผู้เขียนใช้ SideKick เป็นตัวเขียนโปรแกรมในภาษาต่าง ๆ หลายภาษาที่ใช้แยกในการคอมไพล์ เพราะง่ายและสะดวกมาก ใน การออกจากโปรแกรมและเข้าโปรแกรม ในที่นี้ผู้เขียนจะกล่าวถึงวิธีการใช้ Editor Sidekick เพื่อเป็นแนวทางในการใช้ใช้ทำโปรแกรม หรือ สำหรับผู้ที่มีความถนัดในการใช้ Editor ตัวอื่นก็ขอให้ใช้ Editor ตัวที่ถนัดไปก็แล้วกัน หรือจะอ่านไว้เพื่อประดับความรู้ของผู้อ่านเองก็เป็นสิ่ง ที่ดี สำหรับ Sidekick ทางผู้เขียนเองจะขอกล่าวไว้เท่าที่จำเป็นในการเขียนโปรแกรมเท่านั้น ถ้าหากว่าผู้อ่านต้องการที่จะศึกษา ค้นคว้าไปมากกว่า นี้ก็สามารถหาซื้อคู่มือการใช้ได้ตามท้องตลาด หรือร้านขายหนังสือทั่ว ๆ ไป
