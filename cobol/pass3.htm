Identification Division
          เป็นส่วนหนึ่งหรือเป็นส่วนแรกเริ่มของโปรแกรมภาษาโคบอล ซึ่งจะต้องมีเสมอ สำหรับภายในรายละเอียดของ Division นี้จะมีหรือไม่มีก็ได้ เมื่อเริ่มต้นเขียนโปรแกรม โปรแกรมเมอร์จะต้องเขียนชื่อ Division นี้ก่อนเสมอและจะเริ่มต้นที่ Area A เสมอ

ใน Division จะมีสองระดับ คือ ส่วนที่เป็น Division คือ Identification Division และส่วนที่เป็น Paragraph ดังนี้คือ

  • Program-Id Paragraph ที่จะต้องเขียนในทุก ๆ โปรแกรมเพื่อเป็นการบอกให้ Operating System รู้ว่าขณะนี้กำลังทำโปรแกรมอะไรอยู่เพื่อประโยชน์ในการติดตาม และ ควบคุมการทำงานของโปรแกรมนี้ตลอดทุกขั้นตอน โดยจะต้องตั้งชื่อโปรแกรม (Program ID ) ตามหลักเกณฑ์
  • Author Paragraph เป็น Paragraph ที่มีไว้เพื่อใช้ระบุวื่อโปรแกรมเมอร์ ที่เขียนโปรแกรมนี้ และเป็นประโยชน์ในการติดต่อประสานงานร่วมกันเป็นทีม
  • Installation Paragraph เป็น Paragraph ที่ใช้อ้างอิงหรือเป็นหลักฐานระบุหน่วยงานที่รับผิดชอบ หรือที่เกี่ยวข้อง หรือชื่อหน่วยงานที่จัดทำโปรแกรมนี้ขึ้นมา
  • Date-Written Paragraph เป็น Paragraph ที่ระบุถึงวันที่ทำโปรแกรมนี้ขึ้น
  • Date-Compiled Paragraph เป็น Paragraph ที่ใช้ในการระบุถึงวันที่ทำการ Compile ซึ่งหลังจากที่ทำการ Compiler จะใส่วันทีที่ทำการ Compile ให้เอง
  • Security Paragraph เป็น Paragraph ที่ใช้แสดงรหัสผ่านสำหรับผู้ที่มีสิทธิ์ใช้โปรแกรมนี้เท่านั้น จุดมุ่งหมายของโปรแกรมนี้อาจจะต้องการรักษาความลับหรือความปลอดภัยของโปรแกรม ซึ่ง Paragraph นี้จะอธิบายให้ผู้ใช้ หรือผู้อ่านได้ทราบต่อไป
Identification Division
           เป็น Division ที่สองของโปรแกรม ในบางระบบเราอาจจะละไม่ใส่ Division นี้ก็ได้ หรืออาจจะใส่เฉพาะขื่อของ Division นี้เท่านั้นก็ได้ ถ้าหากว่าโปรแกรมนี้ไม่มีการใช้แฟ้มข้อมูลที่เป็น Input Output เลย Environment Division ประกอบด้วย 2 Section ด้วยกัน
  • Configuration Section เป็นส่วนที่ใช้เขียน หรือระบุรายละเอียดของเครื่องคอมพิวเตอร์ที่จะใช้ในการ Compile และการปฏิบัติงาน(Execute) ตามคำสั่งในโปรแกรม โปรแกรมเมอร์อาจจะละเว้นไม่เขียนเอาไวัภายในโปรแกรมก็ได้ เพราะโดยทั่วไปโปรแกรมที่ถูกสร้างขึ้นจะถูกนำมาทดสอบ และสังทำงานทันทีอย่างต่อเนื่องภายในเครื่องเดียวกันดังนั้นภายใน Section นี้มีไว้เพียงเพื่อใช้เป็นประโยชน์ในการอ้างอิง ซึ่งใน Section นี้จะประกอบไปด้วย 2 Paragraph
    • Source-Computer ระบุถึงเครื่องคอมพิวเตอร์ที่ใช้ในการเขียนโปรแกรมขึ้นมาและทำการ Compile ให้เป็น Code Program ชนิด Object Code
    • Object-Computer เป็นการระบุถึงเครื่องคอมพิวเตอร์ที่ใช้กับ Object Program โดยส่วนใหญ่จะใช้เครื่องคอมพิวเตอร์ที่เป็นชนิดเดียวกันกับเครื่องคอมพิวเตอร์ที่ใช้ในการแปลเป็น Object Code หรือใช้เครื่องฯ เดียวกันนั่นเอง
  • Input-Output Section ในส่วนนี้โปรแกรมเมอร์จะต้องเขียนในกรณีที่มีการเรียกแฟ้มข้อมูลเข้ามาใช้ หรือในกรณีที่มีการส่งผลลัพธ์ที่ได้จากการทำงานของโปรแกรมไปเก็บไว้ในแฟ้มที่สำหรับเก็บผลลัพธ์ หรือมีการสร้างแฟ้มข้อมูลชั่วคราวขึ้นมาใช้ในการทดสอบโปรแกรมต่อถ้าไม่มีการใช้แฟ้มข้อมูลในส่วนของ Section นี้ก็ไม่ต้องระบุใน Section นี้ยังสามารถแยกออกไปได้อีก 2 Paragraph ดังนี้
    • File-Control Paragraph
    • I-O-Control Paragraph
